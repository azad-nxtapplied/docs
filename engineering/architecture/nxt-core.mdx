## Design principles

<AccordionGroup>
  <Accordion title="Depend downward only" icon="arrow-down">
    Tiers form an acyclic DAG. Lower components stay "mechanical" — they read `workspace_id` from ambient Context rather than resolving the tenant themselves — so there are no cycles.
  </Accordion>
  <Accordion title="Group by bounded context" icon="object-group">
    Identity + Tenancy + Authorization → IAM. Config + Secrets together. Context + Observability cross-cutting. Industry-standard groupings, not one-service-per-concept.
  </Accordion>
  <Accordion title="Tenant isolation everywhere" icon="shield">
    Every request is auto-scoped to its workspace — app-level filter + DB row-level-security backstop; cache via a workspace key prefix.
  </Accordion>
  <Accordion title="Types flow up, never down" icon="arrow-up">
    A thin shared kernel holds universal primitives (Context, IDs, base Event). Each component owns and exports its own contract DTOs; subtypes resolve via a `type` discriminator on the wire (CloudEvents-style), never runtime class identity.
  </Accordion>
  <Accordion title="Authorization vs Policy Gate" icon="key">
    IAM gates _capability_ (do you have a licence). The Policy Gate gates the _specific action_ at execution time. Both can deny.
  </Accordion>
</AccordionGroup>
