The single place a component asks what the setting is for this workspace and gives me this credential. **General platform config only** — feature flags, operational toggles, connector creds, signing keys. **Not** business/domain config (playbook taxonomy lives with the playbook, authorize/deny rules with the Policy Gate, case data in AI Memory).

## Contract

| Method | What it does |
| --- | --- |
| `get` | Read a setting, flag, or secret for the current workspace (an `isSecret` flag marks secrets for safe handling) |
| `set` | Create or update a setting, flag, or secret (used by Admin / governance) |

## State & storage

Owns per-workspace settings + feature flags, plus secrets kept encrypted at rest. A workspace value overrides a platform-wide default. Source of truth for secrets is an external secret manager; a synced encrypted copy keeps reads fast. Secret values are never logged.

## Dependencies

Reads/writes through [Data Access](/engineering/architecture/components/data-access) (Tier 0), plus [Context & Observability](/engineering/architecture/components/context-observability). The secret-manager sync runs out-of-band — not a runtime dependency, so this stays acyclic.

## Infra & deployment

Relational DB (via Data Access) for settings + encrypted secrets, a cache for hot config reads, an external secret manager as source of truth. In-process library; config changes take effect at runtime with short-lived cache invalidation — no redeploy.
